# Security Policy

Ghostshift is expected to handle development metadata, task traces, and eventually sensitive repository context, so security needs to be part of the project from the beginning.

## Reporting A Vulnerability

Please do not open public issues for security vulnerabilities. Use the repository host's private reporting channel once available.

## Security Priorities

- local data should stay local by default
- exports should be explicit, never surprising
- plugin boundaries should be documented and narrow
- self-hosted deployments should not depend on hidden services
